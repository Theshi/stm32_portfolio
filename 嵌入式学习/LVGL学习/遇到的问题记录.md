解决方法如下：
**C/C++混合编程导致的链接错误**
==分析：==问题的根源在于，图片数据是作为C文件（`.c`）编译的，但被作为C++文件（`.cpp`）编译的 `main` 函数或其他C++代码调用。C++编译器为了支持函数重载等特性，会修改函数名，导致链接器找不到C语言风格定义的符号。
==方案：==
**为图片变量添加 `extern "C"` 声明**
```
// 使用 extern "C" 告诉 C++ 编译器，这些符号是 C 语言的
extern "C" {
    LV_IMG_DECLARE(GitHub_Logo);
    LV_IMG_DECLARE(img_lvgl_logo);
}
```
